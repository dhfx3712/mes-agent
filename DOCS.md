## 2. 引用规范

### 2.1 跨文档引用

当文档 A 需要引用文档 B 的内容时，**不得复制内容**，必须使用引用语句：

```
> 架构模式详情见 `ARCHITECTURE.md`「架构模式」章节。
> 执行流程详情见 `AGENTS.md`。
> 工具清单详情见 `TOOLS.md`。
```

### 2.2 引用格式

```
> {内容概要}见 `{文件名}`{可选：章节名}。
```

### 2.3 禁止行为

- ❌ 禁止在 A 文档中复制 B 文档的核心内容
- ❌ 禁止在 TOOLS.md 中写架构模式表格
- ❌ 禁止在 AGENTS.md 中写工具清单
- ❌ 禁止在 SOUL.md 中写路由规则或架构内容

## 3. 修改约束

### 3.1 新增文档

新增文档必须：
1. 在 DOCS.md 的职责矩阵中注册
2. 明确「包含内容」和「不包含内容」
3. 不得与现有文档职责重叠

### 3.2 修改现有文档

修改文档时：
1. 检查修改内容是否属于本文件的职责范围
2. 如果内容属于其他文档 → 修改对应文档，本文件改为引用
3. 如果内容不属于任何现有文档 → 考虑新增文档或在 DOCS.md 调整职责

### 3.3 删除文档

删除文档前：
1. 检查所有引用该文档的文件，更新引用
2. 从 DOCS.md 职责矩阵中移除

## 4. 文档质量要求

| 维度 | 要求 |
|------|------|
| 职责声明 | 每个文档顶部必须有 `> **职责**：` 声明 |
| 引用声明 | 引用其他文档时使用 `>` 格式 |
| 目录 | 超过 20 行的文档应有目录 |
| 格式 | 统一 Markdown，UTF-8 编码 |
| 冲突处理 | AGENTS.md（执行流程）优先级最高，冲突时以 AGENTS.md 为准 |
